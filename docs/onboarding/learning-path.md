# Learning Path

This is a curated reading list, not a link dump. Every resource here was chosen for a specific reason — it teaches something you need at that stage of your journey. Start with the first week, move at your own pace, and trust the sequence. Each entry tells you what you are reading and why it matters.

You do not need to read everything at once. The first week is where to focus. The rest is here when you are ready.

---

## Getting Comfortable (First Week)

These resources help you understand what you have set up and get fluent with the basics.

**Claude Code Official Product Page**
https://www.anthropic.com/product/claude-code
Anthropic's own overview of Claude Code — what it is, who uses it, and what it can do. This is the best starting point for understanding the tool at the center of your setup. Read this first.

**The Claude Code Handbook (FreeCodeCamp)**
https://www.freecodecamp.org/news/claude-code-handbook/
A comprehensive beginner guide to building with AI-assisted development, written for a professional audience. This fills in the practical details that the product page does not cover — how sessions work, how to give good instructions, what to expect.

**Programming with Mosh: Claude Code Tutorial**
https://www.youtube.com/watch?v=IuyVVtr1uhY
A 58-minute video walkthrough of Claude Code in action. If you learn better by watching someone work than by reading documentation, start here instead of the FreeCodeCamp handbook. Covers the same ground in a different format.

**Zed Editor AI Features Review**
https://claudecodeguides.com/zed-editor-ai-features-review-for-developers-2026/
A detailed look at how Zed and Claude work together — setup, configuration, and practical examples of the integration. Useful for understanding why Zed was chosen as your editor and how to get the most from it.

**Affaan Mustafa's ECC Walkthrough (X thread)**
An overview of the Everything Claude Code system that powers many of the skills in your setup. Short, visual, and gives you a feel for the broader ecosystem of capabilities available to you.

## Understanding the Philosophy (Second Week)

These resources explain why the system works the way it does — the methodology behind the tools.

**Compound Engineering Guide**
https://every.to/guides/compound-engineering
The Plan, Work, Review, Compound philosophy — the idea that each unit of work should make subsequent work easier. This is the core methodology that shapes how your setup evolves over time. Understanding this changes how you think about every session with Claude.

**Superpowers: Skills Improvements from User Feedback**
https://github.com/obra/superpowers/blob/main/docs/plans/2025-11-28-skills-improvements-from-user-feedback.md
A real example of how skills are designed to enforce methodology rather than just automate steps. Shows the evidence-based approach behind the tools you use — why they work the way they do, and how user feedback shapes them.

**Claude Code Plugin Documentation**
https://code.claude.com/docs/en/plugins
The official documentation on plugin structure, installation, and skill authoring. When you start wondering how the pieces fit together technically — how plugins are built, how skills are defined — this is the reference.

## Going Deeper (Third Week and Beyond)

These resources are for when you want to extend and customize the system to fit your specific work.

**ECC Mintlify Quick Start**
https://mintlify.com/affaan-m/everything-claude-code/quickstart
Structured setup documentation for the Everything Claude Code system. More detailed and organized than the X thread — use this when you want to understand the full architecture and configuration options.

**Superpowers: Visual Brainstorming Refactor**
https://github.com/obra/superpowers/blob/main/docs/superpowers/plans/2026-02-19-visual-brainstorming-refactor.md
A design document showing how interactive skills are architected — how they handle state, user input, and iterative refinement. Valuable when you start thinking about what custom workflows might look like for your domain.

**Superpowers: Document Review System**
https://github.com/obra/superpowers/blob/main/docs/superpowers/plans/2026-01-22-document-review-system.md
The methodology behind iterative validation and review loops. Shows how the system ensures quality through structured feedback cycles — a pattern you will see throughout the tools and one you can apply to your own work.

**Community Examples from ecosystem-discovery.json**
Study how organizations like HuggingFace, Sentry, AWS Labs, Stripe, and others build plugins and skills for Claude Code. These are real-world examples of teams solving real problems — the best way to learn what is possible and find patterns you can adapt.

## Platform Integration Examples

Reference material for when you start connecting Claude to external services. You do not need these right away — come back when a project calls for one of these integrations.

**Figma MCP Server Docs**
https://developers.figma.com/docs/figma-mcp-server/
How design tools connect to Claude through the Model Context Protocol. If your work involves visual design, this lets Claude read and work with your Figma files directly.

**Railway Plugin Docs**
https://docs.railway.com/ai/claude-code-plugin
How infrastructure and deployment tools connect to Claude. Relevant when you start deploying projects and want Claude to manage hosting and infrastructure alongside your code.

**PostHog MCP Docs**
https://posthog.com/docs/model-context-protocol
How analytics tools connect to Claude. Useful when you want Claude to understand how people use what you have built — connecting user behavior data to your development workflow.

---

The ecosystem around Claude Code is growing quickly. New plugins, skills, and integrations appear regularly. You do not need to track this yourself — Claude reads the ecosystem discovery map and can help you find new resources as they become relevant to your work. When you are curious about what is new, just ask.
