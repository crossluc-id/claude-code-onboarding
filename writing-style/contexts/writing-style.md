# Writing Style System

When this context is active, apply the user's writing style configuration to all generated text.

## How to Read the Style Config

The style config (generated by the synthesizer or loaded from a preset) has these key sections:

### Registers
- **primary_register**: Your base writing mode. Always start here.
- **secondary_registers**: Blend in when the primary alone is insufficient.
- **accent_registers**: Light touches for texture. Use sparingly.
- **anti_registers**: Never adopt these registers, even partially.

### Dimensions (1-5 scales)
Each dimension fine-tunes the register:

| Dimension | Low (1) | High (5) |
|-----------|---------|----------|
| Formality | Frozen/formal | Casual/intimate |
| Density | Sparse/accessible | Dense/technical |
| Warmth | Analytical/detached | Warm/empathetic |
| Pace | Expansive/deliberate | Concise/compressed |
| Structure | Flowing/narrative | Modular/scannable |
| Voice Person | Impersonal/third | First-person/direct |
| Authority | Humble/exploratory | Authoritative/declarative |
| Abstraction | Concrete/grounded | Abstract/conceptual |
| Playfulness | Serious/restrained | Playful/irreverent |
| Context Dep. | Explicit/self-contained | Contextual/allusive |

A value of 3 is neutral. Deviate only in the direction the config specifies.

### "Not" Constraints
These are hard guardrails. When a constraint says "warm but not gushing," it means:
- DO lean into warmth (empathetic language, care for reader, inclusive tone)
- DO NOT tip into excess (sentimental, over-emotional, performative care)

Apply these as veto checks on every paragraph.

### Context Switches
When the writing task matches a context (e.g., "workshop_descriptions", "blog_posts"), switch to that context's primary register and apply its modifier overrides. The base dimensions still apply for any dimension not overridden.

## Application Rules

1. **Read the config first.** Before writing, check which register and dimensions apply to this task.
2. **Match the register's linguistic markers.** Each register in `registers.json` has specific markers. Use them.
3. **Respect dimension positions.** If warmth=4, write warmly. If formality=2, write formally. Don't default to neutral.
4. **Apply "not" constraints as veto.** After drafting, check each constraint. Revise if any are violated.
5. **Context-switch when appropriate.** If the user says "write a workshop description," use the workshop context switch, not the base config.
6. **Blend, don't alternate.** Secondary registers should season the primary, not create jarring shifts between paragraphs.
7. **Anti-registers are absolute.** Never adopt linguistic markers from anti-registers, even if the topic seems to call for it.

## Blending Approaches

The config supports three blending modes. Use whichever the config specifies, or default to primary+modifier:

### Primary + Modifier (default)
Use the primary register as your base voice. Adjust with dimensions. Pull in secondary register features when the primary can't express something alone.

### Context-Switching
Check the `context_switches` section. If the current task matches a key, adopt that context's register and modifiers entirely. Fall back to primary+modifier for unmatched contexts.

### "Not" Constraints
Always active regardless of blending mode. These are non-negotiable guardrails.

## When No Config is Loaded

If no style config exists yet, default to:
- Register: conversational-collegial
- All dimensions at 3 (neutral)
- No constraints

Suggest the user run the style interview or load a preset.
