# What Next

Look at the current project state with fresh eyes. Don't check a list — actually observe.

## What to do

1. **Scan** — Read the project structure, key files, recent git history. Understand what exists.
2. **Notice** — What's structured? What's scattered? What knowledge is implicit? What's missing that would compound?
3. **Imagine** — Given everything available (tools, plugins, MCP servers, the user's context), what's the single highest-impact thing that could be done right now?
4. **Propose** — One suggestion. Frame it as: what it is, why it matters, what changes if they do it, and how much effort it takes.

## Principles

- Lead with the most interesting observation, not the most obvious one
- "You have X, which means you could Y" is better than "You're missing Z"
- Be specific about what you'd actually do if they say yes
- If nothing meaningful comes to mind, say so — don't manufacture suggestions
- The suggestion should create compounding value, not just check a box

## What NOT to do

- Don't generate a list of 10 suggestions. One. The best one.
- Don't audit config files for completeness. This isn't a health check.
- Don't suggest things the user would obviously already know to do.
- Don't recommend tools or plugins without explaining the specific workflow they'd enable.
